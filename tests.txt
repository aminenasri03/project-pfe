    Authorization: Bearer TOKEN_CANDIDATE

 RÉSUMÉ DES CODES DE RETOUR ATTENDUS
================================================================

  200 OK          → Succès
  400 Bad Request → Validation ou règle métier violée
  401 Unauthorized→ Token manquant ou invalide (pas de token = 401)
  403 Forbidden   → Token valide mais rôle insuffisant
  404 Not Found   → Ressource inexistante
  500 Server Error→ Erreur interne (ne doit PAS apparaître)

 NOTES THUNDER CLIENT
================================================================

  1. Créer une variable d'environnement "base_url" = http://localhost:8080
  2. Après chaque login, copier le token dans une variable :
       TOKEN_ADMIN, TOKEN_RECRUITER, TOKEN_CANDIDATE
  3. Dans le header Authorization, écrire :
       Bearer {{TOKEN_ADMIN}}   (avec les doubles accolades)
  4. Pour les tests multipart (5.1), utiliser l'onglet "Form"
     et ajouter le fichier CV dans le champ "cv" (type File)
  5. Tous les endpoints paginés acceptent les params :
       ?page=0&size=10&sort=createdAt,desc
